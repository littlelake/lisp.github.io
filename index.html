驾考科目二关于倒车入库

发表于 2018-03-23 | 分类于 生活

汽车发动前的准备工作

  1. 在打火前首先向教练问好，然后调整好座椅水平和垂直的位置，让自己处在一个舒服的位置上；
  2. 调整左右的后视灯，左边的后视灯调整为让自己坐着时能看到左后方的车轮，右边后视镜也同样尽量让自己能看到右方后面的车轮；
  3. 打火之前自己去试一下离合器，看是否有什么问题，同样也是让自己能找到点感觉。

在倒车入库中遇到的问题

  1. 离合器的掌握（这个是重点），让汽车能尽量保持缓慢而且匀速；
  2. 当汽车偏离入库标线一点是，要用方向盘去进行补救。主要会遇到的情况有以下：（1）如果是从右边倒车入库，右边后视镜显示右边比较宽，那么方向盘就向顺时针打；（2）同样是从右边入库，如果倒车时右边进去车与标线不是平行，而是里面比外面窄，那么这个时候方向盘向逆时针打；（3）汽车从左边倒车入库，如果后视镜显示左边离标线的距离比较宽，那么这个时候方向盘往逆时针方向打；（4）在左边倒车入口如果左边车进去后和标线的距离是外面比里面宽，那么方向盘往顺时针方向打。

划重点

离合器控制，方向盘转的速度

搭建hexo平台遇到的坑

发表于 2018-03-22 | 分类于 前端

创建项目仓库

创建项目仓库的时候我是直接用的lisp，但是后面hexo d的时候会报错，显示当前仓库不存在。后面查了资料后知道要这样创建才行githubname.github.io

source下创建一个CNAME文件（注意没有.txt后缀）

创建CNAME的目的是为了有一个自己的个性化域名

配置_config.yml

在hexo d提交github时一直提示提交不成功，后面查看了一下原因是_config.yml中deploy的配置问题。网上的配置是

deploy:
  type: git
  repo: ssh://git@github.com:littlelake/lisp.github.io
  branch: master

或者

deploy:
  type: git
  repo: git@github.com:littlelake/lisp.github.io.git
  branch: master

其实主要是在deploy.repo这块出现了问题，后面我把ssh://和.git不要了，这个时候就可以提交成功了。

修改主题

修改主题我是用的git clone https://github.com/iissnan/hexo-theme-next themes/next，但是克隆下来之后next主题不是在themes，而是在theme，所以后面参考了网上的解答之后说是将theme下面的next文件夹拖到themes中，然后在_config.yml中修改theme字段。然后就没有问题了。

常用命令

hexo g      // 生成静态文件
hexo s      // 本地运行
hexo d      // 部署到github上
hexo d -g   // 生成静态文件并部署到github上
